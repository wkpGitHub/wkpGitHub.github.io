性能和兼容性说明：

  【html性能和兼容：】

  html部分只有图片加载上遇到性能问题，
  特别是在移动端，网速相对pc端较慢；采用图片懒加载方式，滑动到
  相应的位置，开始加载显示相应位置的图片；
  轮播大图采用css背景图片的写法，让它优先加载；

  兼容上，只有placeholder属性，需要js方法兼容低版本的浏览器；



  【css兼容和性能：】

  样式分公用样式和私有样式：
  公用样式写在common.css里面；

  私有样式，写在具体的对应的css文件里，私有样式我一层层套的很紧密，
  以防后期改样式，会不知不觉改动到其他页面；

  css我能兼容到ie7+；
  css3不兼容ie7和ie8，我会分别用ie的hack写法，采用另一种样式做兼容；
  背景图标采用雪碧图，防止鼠标经过闪屏；

  2)返回接口，可以提高其他js文件调用：models.autoHeight()；

  3)采用惰性模式，提高性能，例如：

  //事件绑定兼容，通常程序员写法：
  var addEvent = function (dom, type, fn) {
    if (window.addEventListener) {
      dom.addEventListener(type, fn, false);
    } else if (window.attachEvent) {
      dom.attachEvent('on' + type, fn);
    } else {
      dom['on' + type] = fn;
    }
  }

  //我的高性能写法：
  var addEvent = (function (dom, type, fn) {
    if (window.addEventListener) {
      return function (dom, type, fn) {
        dom.addEventListener(type, fn, false);
      }
    } else if (window.attachEvent) {
      return function (dom, type, fn) {
        dom.attachEvent('on' + type, fn);
      }
    } else {
      return function (dom, type, fn) {
        dom['on' + type] = fn;
      }
    }
  })();


  4)采用函数节流方式提高性能，例如：

  //窗口大小改变时，调整<div id="J_height"></div>的高度
  function economize(methods) {
    clearTimeout(methods.tId);
    methods.tId = setTimeout(function () {
      methods();
    },100)
  }

  addEvent(window, 'resize', function () {
    economize(function () {
      var _height = document.documentElement.clientHeight;
      document.getElementById('J_height').style.height = _height + 'px';
    });
  });

  工作中遇到一次性实例化几百个下拉框对象实例，在低版本ie下，需要10秒左右，导致页面卡顿；
  我想到两种解决：
  1、采用瀑布流方式，滚动到底部再分页加载的方式；
  2、先不实例化下拉框对象；鼠标经过哪一个下拉框，再实例化（鼠标经过在前面，点击在后面）；
     这样点击的时候，已经实例化好了，可以用了；

能为贵公司做什么：

  【把浏览器交给前端，把服务器留给后端】
  1、移动端页面：
  【移动端上弃绝媒体查询不精准的做法】
  采用获取屏幕宽度的方法，用js设置rem单位的大小；配合百分比，
  做到和设计稿高匹配的精致页面；随着屏幕大小的变化，里面的页面结构(字体大小、图片宽高，等等)会
  实时的发生等比例弹性变化；

  2、pc端页面：
  1)响应式自适应页面：
  采用css媒体查询再引入两个js做法，让低版本浏览器也兼容媒体查询功能；

  2)页面渲染、数据可视化和iframe页面嵌套
  给我相应的请求接口和接口API说明，其他的页面渲染(dom元素操作)交给我来完成；
  数据可视化，我采用的是echart2插件
  熟练iframe页面嵌套，js不同层次页面元素获取交互；

  3、pc端和移动端兼容响应式网站（不是用bootstrap）
  第一种做法：js判断用户代理，确定设备类型，跳转相应的域名；
  第二种做法：js判断用户代理，确定设备类型，引入相应的css文件和js文件；

  4、关于微信小程序和Angular、vue、react；
  微信小程序在公司做过具体的一个泰国接送机项目；
  Angular、vue、react；对于Angular2涉及Es6，

  5、其他js框架：
  在大华股份有限公司工作，其中js框架大半都是自己封装的，还有在其他框架上进一步封装的，我在
  工作之余时间，其中2000行左右的js框架，我会深入源码，自己写出API和具体代码运作说明，留给自己以后用工作用；
  习惯查看源码，了解框架的内部是如何嵌套联系的；过于复杂的上万行的js大框架，我表示没有那个精力
  和能力全面解析出代码逻辑；有时候也自己封装一些小插件（播放时间轴、select下拉等等）

  6、工作效率和处理问题能力：
  1)工作效率：
  可能因为是设计出身的，对于ps异常熟练；对于获取宽高和字体大小的值速度快；
  结合一些其他的窍门；在几家公司中，都是编辑页面速度最快的(是快出很多，1.5倍以上)；

  2)处理问题能力：
  在小公司工作只有自己一个前端，遇到问题只能独立处理，自己查找资料，换位思考换一种觉得方案；
  这样的好处是能一个人独当一面，坏处比较耗时；

  在大公司是一个前端团队，遇到问题可以相互请教；好处是工作效率高，坏处是少了点思考；当然遇到
  新问题，还是要自己独立思考，查找资料；

  7、业务逻辑问题
  一个成熟的前端不仅仅是会敲代码，而且会一开始审视设计稿，深入后端思考问题，
  提出操作逻辑和业务逻辑不合理或者不可行的地方。以免最后到了后端才发现问题，
  避免又从设计改设计稿开始重新做，然后又到前端又到后端一条线的工作量。
